# 블로깅용 MD 문서 생성 가이드

사용자가 "블로깅용 MD문서 생성"이라고 입력하면, 그날 학습한 내용을 아래 형식에 맞춰 마크다운 문서를 생성한다.

## 작성 조건

- **독자 수준**: 브라우저 API를 처음 접하는 프론트엔드 개발자
- **분량**: 각 섹션당 핵심만 간결하게
- **파일명**: `src/days/dayNN/dayNN-blog.md` 형식으로 저장

## 생성 규칙

- 대화 중 학습한 내용(질문, 답변, 코드)을 모두 참고하여 작성
- 코드 예시는 해당 day의 학습 파일(`src/days/dayNN/`) 코드를 기반으로 작성
- "헷갈리기 쉬운 점" 섹션은 반드시 포함하며, 오늘 대화에서 나온 혼동 포인트를 우선 반영
- "답지 파일 기반 내가 작성한 로직 보완할 부분" 섹션은 반드시 포함하며, `*.answer.tsx`와 내가 작성한 `*.tsx`를 비교하여 차이점과 개선점을 구체적으로 작성

## 문서 형식

```markdown
# [DayNN] [학습 주제]

> 배경 지식이 없는 독자도 이해할 수 있도록 작성한 학습 정리입니다.

## 오늘 배운 것 한 줄 요약

[핵심 내용을 한 문장으로]

## 배경: 왜 이게 필요한가?

[이 API/개념이 존재하는 이유, 어떤 문제를 해결하는지]

## 핵심 개념 정리

[개념별로 소제목을 나눠 설명. 비유나 그림(텍스트 다이어그램)을 적극 활용]

## 실제 코드 예시

[학습한 내용을 반영한 코드 스니펫. 주석으로 각 줄의 의미 설명]

## 헷갈리기 쉬운 점

[오해하기 쉬운 개념, 유사 개념과의 차이점, 브라우저 차이, 엣지 케이스 등을 구체적으로 설명]

## 이 기술이 효율적인 케이스

[구체적인 요구사항과 함께 언제 쓰면 좋은지 명시]

## 답지 파일 기반 내가 작성한 로직 보완할 부분

[내가 작성한 코드와 답지 코드의 차이점, 놓친 엣지 케이스나 더 나은 구현 방식, 코드 품질 또는 가독성 측면에서 개선할 점]

## 오늘의 회고

[배우면서 어려웠던 점, 새롭게 알게 된 점, 다음에 더 알아볼 것]
```
